# Фриланс Биржа — README

## Краткое описание

**Фриланс Биржа** — это проект на C++, реализующий платформу для взаимодействия заказчиков и исполнителей. Пользователи регистрируются, выбирают роли (заказчик, исполнитель или обе) и взаимодействуют через систему заказов, предложений и отзывов. Система также включает модуль администратора и встроенный мессенджер.

Основой архитектуры являются классы и их взаимодействие

---

## Основные сущности и их роли

* **User** — базовый класс пользователя (имя, контакты, логин). Позволяет создавать профиль заказчика или исполнителя.
* **Customer** — профиль заказчика (данные, рейтинг, список атрибутов). Может размещать заказы, редактировать их, оставлять отзывы и общаться с исполнителями.
* **Performer** — профиль исполнителя (данные, контакты, документы, выполненные задания, рейтинг). Может откликаться на заказы, выполнять задания, загружать документы и оставлять отзывы.
* **Order** — заказ (описание, цена, сроки, статус, заказчик, исполнитель). Может быть создан, принят, отредактирован или отсортирован.
* **Review** — отзыв (автор, получатель, текст, оценка, дата). Добавляется после завершения заказа.
* **Rating** — рейтинг (числовой балл, среднее значение, количество отзывов). Пересчитывается при добавлении новых отзывов.
* **Messenger** — встроенный мессенджер между заказчиком и исполнителем (сообщения, вложения, даты).
* **Admin** — администратор с правами модерации (работа с отзывами, документами, пользователями и заказами).

---

## Ключевые сценарии

1. **Регистрация** — пользователь создаёт учётную запись.
2. **Выбор роли** — создаётся профиль заказчика или исполнителя.
3. **Размещение заказа** — заказчик публикует Order.
4. **Отклики** — исполнители подают заявки.
5. **Выбор исполнителя** — заказчик выбирает из откликов.
6. **Выполнение заказа** — исполнитель выполняет работу, заказчик принимает или отклоняет.
7. **Отзывы** — стороны оставляют Review, обновляется Rating.
8. **Коммуникация** — при необходимости стороны используют Messenger.
9. **Администрирование** — администратор следит за порядком, проверяет документы и модерирует отзывы.

---

## Статистика и отзывы

* У каждого пользователя формируется рейтинг на основе отзывов.
* Статистика позволяет определить надёжность заказчиков и качество работы исполнителей.

* Поддержка разных типов заказов и дополнительных ролей.

## Запуск
* Создаем папку для сборки
mkdir build
cd build

* Конфигурируем проект
cmake ..

* Собираем
make

* Запускаем
./freelance_exchange

* Для дебага нужно запускать так

mkdir build
cd build
cmake -DCMAKE_BUILD_TYPE=Debug \
      -DCMAKE_CXX_FLAGS="-fsanitize=address -fsanitize=undefined -fsanitize=leak -fno-omit-frame-pointer -g3" \
      ..
make
